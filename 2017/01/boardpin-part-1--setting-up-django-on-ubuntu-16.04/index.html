Boardpin Part 1 : Setting up Django on Ubuntu 16.04

Posted on January 18, 2017

Yes, the name is a play on Pinboard, the bookmarking service created by idlewords (HN username).

What is Boardpin?

Boardpin is a website written in Django that stores and organizes bookmarks. It uses PostgreSQL as the database backend.

I am building Boardpin to learn how to use Django.

I have not decided all the features that I will implement, but the basic idea is this: users signup and they can add/delete/update their bookmarks.

The first step is to install the dependencies (Python 3 and PostgreSQL):

sudo apt-get install python3-pip python3-dev libpq-dev postgresql postgresql-contrib

During installation, PostgreSQL creates a user named postgres. I will login to the postgres shell using this user:

sudo -u postgres psql

Next, I will create a database and associated user for this project:

CREATE DATABASE boardpin;
CREATE USER ubuntu_username WITH PASSWORD 'choose_complex_password';
ALTER ROLE ubuntu_username SET client_encoding TO 'utf8';
ALTER ROLE ubuntu_username SET default_transaction_isolation TO 'read committed';
ALTER ROLE ubuntu_username SET timezone TO 'UTC';
GRANT ALL PRIVILEGES ON DATABASE boardpin TO ubuntu_username;
\q

The database is called boardpin. The database username is the same as the current Ubuntu username. I replaced choose_complex_password with the real password when I executed the command.

I will use a virtual environment for this Django project.

mkdir venv
cd venv
#create venv
python3 -m venv boardpin
#activate
source boardpin/bin/activate

Install Django and psycopg2, which is an adapter used to connect to PostgreSQL.

pip install django psycopg2

Creating the Django project:

cd ../
django-admin.py startproject boardpin

To avoid committing database connection details and the DJANGO_SECRET_KEY variable, I will store these details in ~/.bashrc as follows:

export boardpin_secret_key="SECRET"
export boardpin_db_name="boardpin"
export boardpin_db_user="UBUNTU_USERNAME"
export boardpin_db_password="PASSWORD"

I will read these values in the Django settings.py located in boardpin/boardpin/boardpin. I probably should not have created the outer boardpin directory to hold the readme and LICENSE.

Editing settings.py:

SECRET_KEY = os.environ['boardpin_secret_key']

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.postgresql_psycopg2',
        'NAME': os.environ['boardpin_db_name'],
        'USER': os.environ['boardpin_db_user'],
        'PASSWORD': os.environ['boardpin_db_password'],
        'HOST': 'localhost',
        'PORT': '',
    }
}

It’s a good idea not to commit credentials to git.

To run the project, I changed directory into boardpin/boardpin and ran the following command:

python manage.py runserver

The development server started up at http://127.0.0.1:8000/.

That’s it for today, I will be blogging about learning Django by building Boardpin, so stay tuned for the next post.
